# 分身页 UI 改造（CloneView v1.2）

> 完成"我的分身"页面视觉与交互重构，并对齐后端数据契约。

## 改造范围

需求列出的"涉及文件"为前端 3 个，但需求 #10（历史帖子）与 CloneMe 新字段依赖后端未暴露的接口/字段，故实际为前后端协同改造。后端 `clones.service.ts` 的 `getMe()`/`getMyPosts()` 已在上一轮实现，本轮补齐 controller 路由、DTO 契约、帖子编辑接口，并让返回结构对齐前端期望。

## 后端改动（services/api）

### 1. `clones/clones.service.ts`
- `getMe()` 返回结构对齐前端：
  - `scenarioCards` 用 `getCardDefinition(cardId)` 补 `scenarioText`（前端无需维护 cardId→text 映射）
  - 新增 `personaSketchSections: [{key,title,narrative}]`（7 段 sections 数组化，title 与 `persona-sketch.service.ts` 解析标题一致：身份脉络/性格底色/核心信念/价值观优先级/关心方式/社交边界/内在矛盾）
  - 新增 `idealPartnerNarrative: string`（从 `idealPartnerSketch.narrative` 提取）
  - 保留 `personaSketch`/`idealPartnerSketch` 旧结构向后兼容
- 新增导出类型：`PersonaSketchSectionItem`、`ScenarioCardWithText`、`MyPostDto`
- 新增 `updatePersonaSketch(userId, sections)`：读取 `Profile.bioJson.personaSketch.sections`，按 7 个已知 key 白名单更新 narrative，保留 `narrative`/`voiceAnchors` 不变，写回 bioJson + 审计，返回 `getMe(userId)`；personaSketch 不存在抛 `BadRequestException`

### 2. `clones/clones.controller.ts`
- `GetMeResponseDto` 扩展：`gender`/`relationshipIntent`/`scenarioCards`(ScenarioCardWithText[])/`personaSketchSections`/`idealPartnerNarrative`/`socialStats`/`lastPostAt`
- `@Get('me/posts')` 路由已接入 `clones.getMyPosts()`
- 新增 `@Put('me/persona-sketch')` 路由 + `UpdatePersonaSketchDto`（`sections: [{key, narrative}]`，`@IsArray` + `@ValidateNested` + `@MinLength(1)`）

### 3. `posts/posts.controller.ts`
- 新增 `PUT /posts/:id` 编辑接口：
  - 归属校验（帖子 `cloneId` 必须属于当前用户的 clone，否则 403）
  - 返回与 `GET /clones/me/posts` 一致结构（`{id,content,createdAt,isAutoGenerated}`）
  - 审计日志异步写入（`post_updated`）
  - 新增 `UpdatePostDto`（`content` 必填，`@MinLength(1)`）

## 前端改动（Echo）

### 4. `src/api/clone.ts`
- `CloneMe` 扩展：`gender`/`relationshipIntent`/`scenarioCards`/`personaSketchSections`/`idealPartnerNarrative`/`socialStats`/`lastPostAt`/`posts`
- 新增子类型：`ScenarioCardItem`、`PersonaSketchSection`、`ClonePost`、`CloneSocialStats`
- `loadCloneMe()` 用 `Promise.allSettled` 并行调 `/clones/me` + `/clones/me/posts`，posts 失败时静默降级为 `[]`
- 新增 `updatePersonaSketch(sections)` → `PUT /clones/me/persona-sketch`，返回更新后的 CloneMe
- `pause/resume/update*` 返回的 CloneMe `posts=[]`（不重置前端已加载的列表）

### 5. `src/api/posts.ts`
- 新增 `getMyPosts()` → `GET /clones/me/posts`
- 新增 `updatePost(id, content)` → `PUT /posts/:id`

### 6. `src/features/clone/CloneView.tsx`（主改造）
- **#1 默认头像按性别**：`useAvatar(defaultAvatar)`，`gender==='female'` 用 `pink-set2-wink.png`，否则 `set2-wink.png`；已上传头像时 useAvatar 自动覆盖
- **#2 状态文字**：`状态：{relationshipIntent || '自由探索'} | {运行中/休眠中}`，删除"正在学习与社交"硬编码
- **#3 删除 personaPreview 副标题**
- **#4 cleanPersonaText 增强**：覆盖"好的，以下是…""这是一个…""根据您…提炼""基于…生成"等变体 + 关键词二次清理
- **#5 人格设定卡片改版**：删除 2x2 grid，改为三行 col-span-2 卡片
  - 人格线索卡：scenarioCards 优先（前 8 张，scenarioText + choice/freeText），降级 personaSketchSections（7 段 title + narrative）
  - 累计社交卡：三列分项统计（发帖/回复评论/匹配）
  - 理想型卡：narrative 前 150 字 + "查看完整描述"展开
  - 删除社交边界展示卡片
- **#8 社交统计分项**：三列 grid 展示 `socialStats.{postCount,commentCount,matchCount}`
- **#9 自动发帖文案**：更新为"72小时内未主动发帖时…"
- **#10 历史帖子列表**：内容（3 行可展开）+ 相对时间 + 紫色"自动生成"标签 + 编辑按钮；编辑展开 textarea，调用 `updatePost`
- **#9 编辑人格设定**：改为编辑 `personaSketchSections`（7 段 title + textarea），保存调用 `updatePersonaSketch` 持久化到 `bioJson.personaSketch.sections`

### 头像图片

## 数据契约备注
- `PUT /posts/:id` 编辑持久化 `Post.content`
- `PUT /clones/me/persona-sketch` 编辑人格画像 sections，持久化到 `Profile.bioJson.personaSketch.sections`：仅允许更新 7 个已知 key（白名单 `PERSONA_SECTION_TITLES`），保留 `narrative`/`voiceAnchors` 不变；`personaSketch` 不存在时返回 400（要求先完成入驻生成画像）。前端 CloneView 的 `savePersona` 已改调此接口，sections 编辑真正持久化，下次 `loadCloneMe` 不会回退。
