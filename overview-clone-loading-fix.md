# 分身页"正在苏醒"加载不出 — 根因与修复

## 现象
登录后点击分身页，"分身正在苏醒…"两三分钟不消失。

## 根因（双重，均源自 Upstash Redis 额度耗尽）

Upstash 额度已耗尽：`max requests limit exceeded. Limit: 500000, Usage: 500002`。`.env` 已开 `BYPASS_REDIS=true`，但旁路实现有两个缺陷：

### 缺陷 1：API 进程崩溃（4000 端口无服务）
- `createRedisClient` 创建的 ioredis 实例**没有 `.on('error')` 监听器**
- Upstash 不可达时触发 `error` 事件 → Node EventEmitter 规定 error 事件无监听器则 **throw → 进程崩溃退出**
- `LiveRedisSubscriber` 用 `duplicate()` 创建的实例同样无监听器（duplicate 不继承事件监听器）
- 结果：API 进程反复崩溃，4000 端口无监听，前端连不上后端

### 缺陷 2：getMe 永久 hang（即使 API 在跑）
- `createRedisClient` 用 `maxRetriesPerRequest: null`（BullMQ 兼容配置）
- Upstash 不可达时，`hget` 命令进入 ioredis 重试队列，`maxRetriesPerRequest:null` 让命令**永不 reject**
- `getMe` 的 `await this.redis.client.hget(...)` 永久 pending → try/catch 等不到 → 整个请求挂死
- 前端 `loadCloneMe()` 的 fetch 永远不返回 → "分身正在苏醒"永不消失

## 修复（4 个文件）

### 1. `services/api/src/redis/create-redis.ts` — 根因修复
BYPASS 模式从 `maxRetriesPerRequest:null` 改为 `maxRetriesPerRequest:1`：
- 命令 1 次重试后 reject（约 60ms），**不永久 hang**
- 保持默认 retryStrategy（连接留 `reconnecting` 状态，不 `close`）
- 配合 `lazyConnect:true` + `connectTimeout:1`

> **关键教训**：不能用 `retryStrategy:()=>null` 修复 hang——它让连接进入 closed 状态，触发 `close` 事件的 `flushQueue` 同步抛异常，进程崩溃。

### 2. `services/api/src/redis/redis.service.ts` — error 监听器
构造函数加 `this.client.on('error', ...)`，吞掉 error 事件记 WARN 日志，不崩进程。

### 3. `services/api/src/live/live-redis.subscriber.ts` — duplicate 实例防护
- `duplicate()` 后加 `.on('error')`（duplicate 不继承监听器）
- `subscribe()` 改为 `.catch()`（Node 22 默认 `unhandled-rejections=throw`，reject 会让进程崩）

### 4. `services/api/src/clones/clones.service.ts` — getMe 超时兜底
`hget` 包 `Promise.race` + 2s 超时，防御非 BYPASS 模式下 Upstash 抖动同样挂死请求。

## 验证

### 单元测试（ioredis 行为）
| 配置 | hget 响应 | 连接状态 | 进程 |
|------|-----------|----------|------|
| 原始（maxRetries:null） | **永久 hang** | reconnecting | 不崩 |
| retryStrategy:null | 8ms reject ✓ | closed | **崩（flushQueue）** |
| **maxRetries:1（采用）** | **60ms reject ✓** | reconnecting | **不崩 ✓** |

## 关键设计决策

- **QueueService 用独立 bullConnection**（不受 BYPASS 影响），RedisService.client 仅业务用，可安全改 maxRetries
- `lastPostAt` 在 BYPASS 模式降级为 null（Redis 不可读），前端正常显示"无记录"
- 修复保留 `BYPASS_REDIS` 开关语义：true 时不实际依赖 Upstash，所有 Redis 读取降级为空值
