## 需求与实现对照

### 1. Post 表新增标记（Prisma schema）✅

**文件**：`services/api/prisma/schema.prisma`（Post 模型）

```prisma
isAutoGenerated  Boolean  @default(false) @map("is_auto_generated")
editedAt         DateTime?                @map("edited_at")
metadataJson     Json?                    @map("metadata_json")
```

**迁移**：`services/api/prisma/migrations/20260708000000_add_post_auto_generated_and_edited_at/migration.sql`
- 新增 `is_auto_generated`（默认 false）和 `edited_at`（可空）两列
- 回填逻辑：从 `metadata_json` 优先取显式 `isAutoGenerated` 布尔，缺失时按 `trigger !== 'manual'` 推断，保证历史数据语义一致

### 2. Worker 72 小时定时自动发帖任务 ✅

**调度入口**：`services/worker/src/main.ts`
- 每小时触发一次 `runIdlePostCheck72h`（`setInterval` 60 min）

**核心逻辑**：`services/worker/src/clone-runtime/scheduler.ts` → `runIdlePostCheck72h`
- 查询所有 `status=active` 的 DigitalClone
- 对每个 clone 查最近一条 Post 的 `createdAt`，若距今 > 72h 或从未发帖则触发
- 拉取该 clone 最近一次 AgentSession 的最近 5 条 AgentMessage 作为上下文（speaker 映射为"我/对方"，避免泄露 clone id）
- 入队 `post-draft`（`trigger='idle_72h'`，`context.recentMessages`）

**发帖执行**：`services/worker/src/main.ts` post-draft Worker
- 调 `generatePostContent`（scheduler.ts）走 `composeSystemPrompt` 组装 system prompt，将 recentMessages 格式化为对话片段注入 user prompt
- 写库：`isAutoGenerated = trigger !== 'manual'`（idle_72h → true），`metadataJson = { trigger: 'idle_72h' }`
- 经 moderation 队列审核通过后，由 moderation Worker 更新 Redis `clone:meta:{id}.lastPostAt`

**设计决策**：`lastPostAt` 在审核通过时才更新（而非入队时），避免未审核帖子刷新时间戳导致巡检误判。JSDoc 已记录此约定。

### 3. 帖子编辑接口 PUT /posts/:id ✅

**文件**：`services/api/src/posts/posts.controller.ts` → `updatePost`
- 请求体：`UpdatePostDto { content: string }`（`@MinLength(1)` 校验，trim 后非空）
- 归属校验：帖子 `cloneId` 必须属于当前用户的 clone，否则 403
- 更新 `content` + `editedAt = new Date()`
- 返回与 GET /clones/me/posts 一致的结构（`id / content / createdAt / isAutoGenerated / editedAt`），前端可直接替换列表项
- 审计日志 `post_updated`（异步，不阻塞响应）

### 4. GET /clones/me/posts 返回新字段 ✅

**文件**：`services/api/src/clones/clones.service.ts` → `getMyPosts`
- `select` 含 `isAutoGenerated` + `editedAt`
- `MyPostDto` 接口声明两字段，直接读取专用列（不再从 metadataJson 解析，避免双源不一致）

## 数据流总览

```
每小时 cron
  └─ runIdlePostCheck72h
       ├─ 查 active clones
       ├─ 查最近 post.createdAt（>72h 或无帖 → 触发）
       ├─ 拉最近 AgentSession 5 条消息
       └─ postDraftQueue.add(trigger='idle_72h', context.recentMessages)
            └─ post-draft Worker
                 ├─ generatePostContent（persona + recentMessages → LLM）
                 ├─ post.create(isAutoGenerated=true, metadataJson={trigger})
                 └─ moderationQueue.add
                      └─ moderation Worker
                           └─ 审核通过 → setCloneMeta(lastPostAt)
```
