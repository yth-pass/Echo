## 一、总体评估

### 做得好的地方 🟢

| 维度 | 评价 |
|------|------|
| **入驻对话文案** (`dialogue-copy.ts`) | 口语化、有好奇心、追问有深度。这是整个项目中最"有人味"的部分。 |
| **情境卡片** (Phase1 15 张卡片) | 场景设计精妙，覆盖了道德困境、社交压力、情感纠葛等真实议题。 |

### 主要问题 🟡🔴

| 问题 | 严重程度 | 影响范围 |
|------|----------|----------|
| **错误消息缺乏人格** | 🔴 高 | 全局所有错误态 |
| **加载状态千篇一律** | 🟡 中 | 全局所有加载态 |
| **成功/庆祝时刻缺失** | 🔴 高 | 入驻完成、任务达成等关键时刻 |
| **按钮文案纯功能化** | 🟡 中 | 所有按钮 |
| **空状态过于平淡** | 🟡 中 | 广场、匹配等空态页面 |

| **品牌人格不统一** | 🟡 中 | 不同模块文案风格差异大 |
| **零趣味/彩蛋** | 🟢 低（锦上添花） | 全局 |

## 二、逐类诊断与优化方案

### 2.1 错误消息 —— 🔴 最需要优化的区域

**现状：** 全域错误消息为 `"XXX失败，请检查网络后重试"` 的模板变体，冰冷、机械、没有任何品牌人格。

| 当前文案 | 频次 | 问题 |
|----------|------|------|
| `网络连接失败，请检查网络后重试` | 出现 10+ 次 | 像系统报错，不像朋友提醒 |
| `加载失败` | 6+ ViewModel | 过于简略，未给用户任何信息 |
| `保存失败，请稍后重试` | 多处 | 没有安抚情绪 |
| `发送失败，请检查网络后重试` | 多处 | 无差异，无记忆点 |
| `验证码发送失败，请检查手机号后重试` | 认证模块 | 同样冰冷 |
| `登录失败，请稍后重试` | 认证模块 | 无引导 |
| `孵化失败，请检查网络后重试` | 入驻 Finalize | 关键时刻失败，用户最焦虑 |
| `对话初始化失败，请检查网络后重试` | M4 对话 | 冷冰冰打断对话流 |

**优化建议：** 按错误类型建立三级个性梯度

#### 网络类错误（用户不可控→安抚 + 轻幽默）

| 场景 | 当前 | 建议 |
|------|------|------|
| 通用网络 | `网络连接失败，请检查网络后重试` | `信号溜走了 🌬️ 检查下网络，马上回来。` |
| 登录 | `登录失败，请稍后重试` | `门锁卡了一下，再试一次？` |
| 验证码 | `验证码发送失败，请检查手机号后重试` | `短信好像迷路了 📨 确认下号码对不对？` |
| 发送消息 | `发送失败，请检查网络后重试` | `消息卡在半路了，网络恢复就自动帮你发～` |
| 加载内容 | `加载失败` | `内容还在路上… 下拉刷新，或等它喘口气。` |

#### 服务端错误（服务不可用→诚实 + 安抚）

| 场景 | 当前 | 建议 |
|------|------|------|
| AI 服务暂不可用 | `AI 服务暂时不可用，请稍后重试` | `AI 在打个盹，过几秒再来戳它。` |
| 短信服务暂不可用 | `短信服务暂时不可用，请稍后重试` | `短信通道小堵，稍等一下再试试～` |
| 服务未配置 | `服务未配置，请联系管理员` | `后台还没搭好，我们的工程师正在加急处理。` |
| 登录过期 | `登录已过期，请重新登录` | `太久没动，为了安全需要重新登录一下。` |

#### 孵化/完成类错误（关键时刻→强烈安抚 + 明确路径）

| 场景 | 当前 | 建议 |
|------|------|------|
| 孵化失败 | `孵化失败，请检查网络后重试` | `孵化到一半被风吹跑了… 🥚 别担心，你的数据都在，再试一次就好。` |
| 对话初始化失败 | `对话初始化失败，请检查网络后重试` | `对话还没准备好，再点一次就好。你的问卷答案都还在。` |

### 2.2 加载状态 —— 🟡 中优先级

**现状：** `加载中…` / `处理中…` 几乎覆盖所有场景，完全浪费了用户等待时间的情绪价值。

| 当前 | 问题 |
|------|------|
| `加载中…` | 出现 15+ 次，空白无聊 |
| `处理中…` | 冷冰冰 |
| `提交中…` | 同上 |
| `正在准备对话…` | 还行但可以更好 |
| `分身孵化中…` | 还行但可以更好 |

**优化建议：** 按场景注入微趣味

| 场景 | 建议文案 |
|------|----------|
| 广场加载 | `广场上的人正在赶来…` |
| 匹配加载 | `在茫茫人海中寻找你的同类…` |
| 分身信息加载 | `分身正在苏醒…` |
| 对话加载 | `对话正在连接，对方很快上线` |
| 入驻提交 | `正在把你的故事写进 Echo…` |
| 画像生成 | `正在勾勒你的轮廓…` |
| Finalize 孵化 | `正在把你的人格注进数字世界…` |
| 评论加载 | `评论在路上了` |
| 活动记录加载 | `调取你的社交记忆…` |

### 2.3 成功/庆祝时刻 —— 🔴 最被忽略的区域

**现状：** 整个产品几乎没有庆祝时刻。入驻完成是这个产品的"高光时刻"，用户花了 20-30 分钟填写问卷+对话，最终只有一句：

> `入驻完成` / `已写入人格画像与语言风格，准备进入广场`

这就像跑完马拉松后收到一张 A4 打印纸。

**优化建议：**

#### 入驻完成 → 里程碑庆祝

```
当前：
  "入驻完成"
  "已写入人格画像与语言风格，准备进入广场"

建议（三层递进）：
  1. 视觉层：完整的画像卡片展示（8 节人格画像聚合视图）
  2. 情感层："你的数字分身，已诞生。"
  3. 行动层："去广场让它认识第一个朋友吧 →"
```

#### 入驻各阶段小庆祝

| 阶段 | 当前 | 建议 |
|------|------|------|
| Phase0 名片完成 | `名片已生成，进入下一步…` | `名片就绪 ✨ 进入下一步，说说你是怎样的人。` |
| Phase1 卡片完成 | `完成全部` | `15 张卡片，拼出了一部分真实的你。` |
| Phase1.5 画像确认 | `全部 OK，继续` | `这就是你的画像。它像你吗？` |
| Phase2 对话完成 | `可以结束了` | `聊得不错，你的语气我学会了。孵化去吧 👇` |

#### 其他成功时刻（建议新增）

| 场景 | 建议 |
|------|------|
| 分身发帖成功 | `你的分身在广场说了句话` + 预览卡片 |
| 匹配成功 | 匹配度动画 + `你的分身和 TA 产生了 {n}% 的共振` |
| Handoff 接受 | `缘分跨越了 AI，祝你们聊得开心` |
| 设置保存成功 | Toast: `已保存 ✓`（带微动效） |
| 举报提交成功 | `感谢你的守护，我们会认真处理` |

### 2.4 按钮文案 —— 🟡 中优先级

**现状：** 所有按钮都是纯功能词：`继续` / `发送` / `重试` / `保存` / `取消` / `完成`。

**问题：** 按钮是用户最频繁接触的文字，每次都念同样的字，像在跟 ATM 机交互。

**优化建议：** 选择性注入动感

| 场景 | 当前 | 建议 | 适用原则 |
|------|------|------|----------|
| 入驻前进 | `继续` | `继续` / 保持 | 高频操作不折腾 |
| 入驻完成 | `完成` | `去看看` | 引导探索 |
| M4 发送 | `发送` | `发送` / 保持 | 高频操作 |
| M4 结束对话 | `可以结束了` | `聊够了，去孵化` | 带行动导向 |
| 画像确认 | `全部 OK，继续` | `这就是我` | 强化身份认同 |
| 分身发帖 | `让分身发帖` | `让分身说句话` | 拟人化 |
| 暂停分身 | `暂停分身` | `让分身休息` | 拟人化 |
| 启动分身 | `启动分身` | `唤醒分身` | 拟人化 |
| 重试 | `重试` | `再试一次` | 降低挫败感 |
| 忽略匹配 | `忽略` | `跳过` | 降低负面联想 |
| 拉黑 | `拉黑` | `不再看到 TA` | 功能说明化 |
| 退出登录 | `退出登录` | `退出登录` / 保持 | 安全操作不宜轻浮 |

### 2.5 空状态 —— 🟡 中优先级

**现状：** 部分空状态文案尚可，但缺乏统一的人格。

| 场景 | 当前文案 | 评价 | 建议 |
|------|----------|------|------|
| 广场空态 | `暂无动态，分身发帖后会出现在这里` + `在分身页点击「让分身发帖」，或保持分身运行等待定时发帖` | 功能说明太多 | `广场还静悄悄的。让你的分身说第一句话？` |
| 匹配空态 | `暂无匹配，正在为你寻找缘分` | 还行 | `茫茫人海中，你的分身正在找人。稍等片刻～` |
| 对话空态 | `暂无对话，去广场看看？` | 还行 | `还没人跟你聊上。去广场转转？` |
| 活动记录空态 | `暂无活动记录` | 太干了 | `活动记录还是空的，你的每一步都会被记下来。` |

### 2.6 入驻对话文案 —— 🟢 当前最好，微调即可

`dialogue-copy.ts` 的文案质量已经很高，以下几点可以微调：

| 部位 | 当前 | 建议 |
|------|------|------|
| 开场白前缀 | `嗨～我是 Echo 入驻助手。我会像好奇的采访者一样问你一些问题，用你「...」的口吻随便答就行，不用想太多。` | 过长。建议拆成两次发送：第一条简短打招呼，第二条再说规则。 |
| 收尾 CTA | `聊得不错，你的语气特点我已经记下了。点击下方「继续」，进入分身孵化吧～` | `我记住你了。去孵化你的数字分身吧 👇` |
| 离线兜底重复 | "你问卷里提到典型一天是「...」" 结构在多个兜底中出现 | 可增加兜底变体，避免用户感知重复 |

### 2.7 缺席的「趣味元素」—— 🟢 锦上添花

当前 Echo 项目完全没有以下趣味元素：

| 元素 | 说明 | 简单实现方式 |
|------|------|------------|
| **Easter Egg** | 隐藏彩蛋 | 连续点击 Logo 5 次触发彩虹模式 |
| **Konami Code** | 经典键盘彩蛋 | `↑↑↓↓←→←→BA` 切换趣味主题 |
| **入驻进度拟人化** | 进度条不只是百分比 | 用"人格碎片收集度"替代"50% 完成" |
| **节日/季节性文案** | 时令感 | 春节/七夕时广场横幅换一句话 |
| **错误页 404 趣味化** | 页面不存在 | `这个页面跟你的前任一样消失了。回广场吧。` |

### 2.8 Android 端硬编码问题 —— 🟡 中期应解决

**现状：** 6+ 个 ViewModel 中重复出现 `"加载失败"` 硬编码，Tab 标签在 `EchoNavHost.kt` 中硬编码，推送通知文本在 `EchoFirebaseService.kt` 中硬编码。

**建议：** 将所有重复硬编码抽入 `strings.xml`，与 Web 端建立统一的文案映射表。

## 三、品牌人格一致性建议

当前 Echo 的文案在三个极端之间跳跃：

```
冰冷功能性 ──────────────────────────── 过度口语化
   ↑                                          ↑
 "加载失败"                            "嗨～不用客气，就当跟朋友聊天"
 "网络连接失败，请检查网络后重试"       "你的语气特点我已经记下了"
 "服务未配置，请联系管理员"             "用你平时说话的口吻就行"
```

**建议建立三层文案人格梯度：**

| 层级 | 适用场景 | 人格 | 示例 |
|------|----------|------|------|
| **暖** | 入驻对话、成功庆祝、空态引导 | 朋友般的好奇 + 鼓励 | `你的画像已经出现了第一笔` |
| **温** | 加载态、按钮、提示、表单标签 | 可靠的伙伴 | `广场上的人正在赶来…` |
| **稳** | 错误态、授权书、账号安全、支付 | 专业但不说教 | `为了安全需要重新登录一下` |

### Echo 品牌文案人格卡

```
品牌人格：一个"懂你的 AI 社交伙伴"
- 像朋友一样说话，但不肉麻
- 像助手一样可靠，但不冰冷
- 像采访者一样好奇，但不冒犯
- 偶尔幽默，但从不轻浮
```
