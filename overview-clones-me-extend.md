# Overview: GET /clones/me 扩展 + 新增 GET /clones/me/posts

## 目标

让前端从单一 API 拿到分身页需要的所有数据，避免多次请求拼装。

## 改动文件

| 文件 | 改动 |
|------|------|
| `services/api/prisma/schema.prisma` | Post 模型新增 `metadataJson Json?` 列 |
| `services/api/prisma/migrations/20260707000000_add_post_metadata_json/migration.sql` | 新增迁移：`ALTER TABLE posts ADD COLUMN metadata_json JSONB` |
| `services/worker/src/main.ts` (L227) | post.create 时写入 `metadataJson: { trigger, isAutoGenerated }` |

| `services/api/src/clones/clones.controller.ts` | GetMeResponseDto 扩展 + 新增 `GET /clones/me/posts` 端点 |

## GET /clones/me 新增返回字段

| 字段 | 类型 | 来源 |
|------|------|------|
| `relationshipIntent` | `string \| null` | `Profile.matchPrefsJson.relationshipIntent` |
| `scenarioCards` | `ScenarioResponse[] \| null` | `Profile.bioJson.scenarioCards`（前 8 张） |
| `personaSketch` | `{ narrative, sections } \| null` | `Profile.bioJson.personaSketch` |
| `idealPartnerSketch` | `{ narrative, dimensions } \| null` | `Profile.bioJson.idealPartnerSketch` |
| `gender` | `string \| null` | `Profile.gender` |
| `lastPostAt` | `string \| null` (ISO 8601) | Redis `clone:meta:{cloneId}.lastPostAt` |

所有新字段 optional（值为 null 时表示数据不存在），完全向后兼容。

## GET /clones/me/posts

```
GET /clones/me/posts
Authorization: Bearer <jwt>
```

返回当前 clone 的所有帖子，按 `createdAt` 倒序：

```json
[
  {
    "id": "uuid",
    "content": "帖子内容",
    "createdAt": "2026-07-06T15:30:00.000Z",
    "isAutoGenerated": true
  }
]
```

`isAutoGenerated` 判断逻辑：
- 从 `Post.metadataJson` 读取
- 如果 `metadataJson.isAutoGenerated` 是布尔值，直接使用
- 否则从 `metadataJson.trigger` 判断：`trigger !== 'manual'` 则为 `true`
- 旧帖子无 `metadataJson` 时默认 `false`

## 设计决策

1. **Post.metadataJson 而非独立列** — 选择 JSON 列而非 `isAutoGenerated Boolean` 列，因为 JSON 更灵活，可同时存储 `trigger`（idle/affinity_boost/welcome/manual/seed）供未来分析用。
2. **Redis 容错降级** — `lastPostAt` 读取 Redis 时 try/catch 包裹，Redis 不可用时返回 null 而非报错。
3. **Profile 查询并行化** — Profile 查询与原有 count 查询合入同一个 `Promise.all`，不增加额外串行延迟。
