## 第三次修复 (2026-06-20 23:42) — 触发条件优化

### Fix 1: session_contact 自动注入
在 agent-turn `turnIndex === 0` 时自动为 A→B 和 B→A 双向注入 `session_contact` 事件。
- 每次 joint session 启动累积 familiarity（+1 per session）
- LLM 不会提取 session_contact，需代码注入
- 位于 main.ts 第 181-203 行

### Fix 2: session 结束兜底触发
当 `turnIndex >= 5`（最后一轮）且 M5/M6 未被 TopicJudge 触发时，兜底运行全管线。
- 提取 M5/M6 管线为 `runM5M6Pipeline()` 局部函数（约 120 行）
- 两个触发点：topic transition (return_to_main/new_main) + session-end fallback
- 日志: `[M6] Session-end fallback triggered` 标识兜底路径
- 使用 `m6Triggered` 标志避免重复执行

### Fix 3: 类型修复
- 为 promote events 添加 `strength: 'moderate'` evidence 字段
- 将 `output` 变量提升至 try/catch 外层作用域

### 验证
- S1-S11 smoke test 全部通过
- E2E 全部通过（4 scenarios）
- Worker TypeScript 编译通过（仅 pre-existing metadataJson 类型警告）
