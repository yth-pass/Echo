# 2026-07-06 工作日志

## Echo 后端三处数据存储缺陷修复

修复用户报告的 3 个数据存储 bug：matchPrefs 不保存、社交统计混算、理想型不可查询。

### 改动文件

1. **`services/api/src/onboarding/onboarding.service.ts`**
   - `submitPhase0`：新增 `buildMatchPrefsJson(dto)` 私有方法，从 `dto.matchPreference` + `dto.goalOnEcho`（作为 `relationshipIntent`）构造 `matchPrefsJson`，在 `Profile.upsert` 时条件写入。无偏好信息时返回 `null` 不覆盖旧值。
   - `finalize`：把 `styleMd` 写入 try-catch 之外，统一同步 `OnboardingSession.surveyJson` 到 `Profile.bioJson`，保证 `Profile.bioJson.idealPartnerSketch` 一定存在（修复 IdealPartnerSketchService 之后 bioJson 可能与 surveyJson 不一致问题）。

2. **`services/api/src/clones/clones.service.ts`**
   - `getMe`：新增 `socialStats: { postCount, commentCount, matchCount }` 分项统计。
   - `matchCount` 实现：用 `agentSession.findMany` 拉所有相关 session，用 `Set<string>` 去重统计不同对方 clone 数（把 `(A,B)` 与 `(B,A)` 视为同一对匹配）。
   - 防御性过滤 `cloneA === cloneB` 的自匹配 session。
   - 保留 `interactionCount` 向后兼容。

3. **`services/api/src/clones/clones.controller.ts`**
   - 新增 `GetMeResponseDto` interface，显式声明返回结构含 `socialStats` 字段。
   - `me()` 方法签名改为 `Promise<GetMeResponseDto>`，让编译器在 service 返回结构变更时报错。

### 验证

- `cd services/api && npm run lint` (tsc --noEmit) 退出码 0，无编译错误。
- 数据库 schema 无变更（`Profile.matchPrefsJson` 列已存在）。
- API 契约向后兼容：旧 `interactionCount` 保留，新 `socialStats` 为新增字段。

### 关键决策

- `matchPrefsJson` 用条件展开 `...(matchPrefsJson ? { matchPrefsJson } : {})` 写入，避免 null 覆盖旧值。
- `matchCount` 选择 "唯一对方 clone 数" 而非 "session 总数"，因为同一对 (A,B) 可能有多条 session 但只算一次匹配关系，符合用户语义。
- `bioJson` 同步放在 finalize 末尾（styleMd 之后），保证拿到的是最新 survey 数据。

## GET /clones/me 扩展 + 新增 GET /clones/me/posts

目标：让前端从单一 API 拿到分身页全量数据。

### 改动

1. **Prisma schema**：Post 模型新增 `metadataJson Json? @map("metadata_json")` 列
2. **迁移**：`prisma/migrations/20260707000000_add_post_metadata_json/migration.sql`（`ALTER TABLE posts ADD COLUMN metadata_json JSONB`）
3. **Worker** `main.ts` L227：post.create 时写入 `metadataJson: { trigger, isAutoGenerated: trigger !== 'manual' }`
4. **Seed** `seed.ts` L113：种子帖子写入 `metadataJson: { trigger: 'seed', isAutoGenerated: false }`
5. **`clones.service.ts`**：
   - 注入 `RedisService`（@Global 无需改 module）
   - `getMe`：Promise.all 并行查询 Profile（gender/bioJson/matchPrefsJson），从 Redis `hget clone:meta:{id} lastPostAt` 读 lastPostAt，解析 bioJson 返回 scenarioCards(前8)/personaSketch/idealPartnerSketch，从 matchPrefsJson 提 relationshipIntent
   - 新增 `getMyPosts(userId)`：查 Post where cloneId orderBy createdAt desc，从 metadataJson 解析 isAutoGenerated
6. **`clones.controller.ts`**：GetMeResponseDto 新增 6 个 optional 字段 + 新增 `@Get('me/posts')` 端点

### 关键设计
- Post 用 `metadataJson Json?` 而非独立 boolean 列（可同时存 trigger 供分析）
- Redis lastPostAt 读取 try/catch 容错降级（Redis 不可用时返回 null）
- 所有新字段 optional（null when absent），向后兼容
- 部署前需 `npx prisma migrate deploy`

### 验证
- `cd services/api && npm run lint` → 退出码 0
- `cd services/worker && npx tsc --noEmit` → 退出码 0
