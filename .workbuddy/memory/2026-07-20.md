### 背景

### 三处 Bug 根因（已通过读代码确认）

**Bug ② Phase 2 状态字段冲突导致死循环（P0 阻断性）**
- 后端 `roleplay-agent.service.ts` startChat 第 117-125 行只复用 `!existingChat.endedAt` 的对话
- chatTurn 第 176 行：`if (chat.endedAt) throw new BadRequestException('对话已结束')`
- chatTurn 第 191-204 行：检测双方告别后自动 `endedAt = now`，返回 `{replies: [], ended: true}`
- 前端 Phase2Roleplay.tsx handleSend 第 320-330 行：看到 `data.ended` 时 setConversation(null)，**但未把 roleId 加入 completedRoles**
- 前端 handleSelectRole 第 226-231 行：loadConversation 命中后直接 setConversation(saved) return，**不调后端 startRoleplay 校验状态**
- loadConversation 第 89 行强制把 status 设为 'active'，与后端 endedAt > 0 永久错位
- 死循环：用户点进对话→发消息→400→退出→角色屏仍显示需要聊→再点进→再 400

**Bug ③ 退回角色屏消息一次性全发 + 气泡数字错误（P1）**
- handleSend 第 343-371 行：API 返回的 N 条 assistant 消息被立刻全部塞进 conversation.messages
- 真正的"逐条显示"靠独立 state `pendingMsgIds` 控制 + setTimeout
- handleBackToRoles 第 467-479 行：清空 pendingMsgIds，但 conversation.messages 已含全部消息且被 localStorage 保存
- 再次进入：loadConversation 拿到 messages，新组件实例 pendingMsgIds 初始为空 → 全部立即显示
- handleBackToRoles 的 setUnreadCount 用 pendingMsgIds.size 覆盖（非累加），且 handleExit 触发 unmount 时不更新 unreadCount

**Bug ① Phase 0/1 退出后字段不恢复（P2）**
- Phase0Identity：formData 初始为 `{}`，完全无 localStorage checkpoint
- Phase1Cards：同设备有 localStorage checkpoint（每 5 卡存），跨设备失效
- OnboardingShell 第 52-66 行：phase 恢复只读 localStorage（useOnboardingSession），不读后端 OnboardingSession
- 后端 onboarding.controller.ts 所有端点都是 POST，**没有任何 GET 进度查询端点**——这是跨设备无法恢复的根因

### 修复方案要点（详见 outputs/onboarding-resume-fix-plan.md）

**Bug ②（P0，4 文件）**：
- 后端 startChat 改为幂等 + 返回 `{status: 'active' | 'ended', existingMessages?, endedReason?}`

**Bug ③（P1，2 文件，依赖 Bug ②）**：
- RoleplayConversation 类型新增 `pendingDisplayIds: Set<string>`，把 pendingMsgIds 独立 state 合并进 conversation
- saveConversation 一起持久化 pendingDisplayIds
- handleBackToRoles 不再清空 pendingDisplayIds，未读数改累加
- unmount 清理 useEffect 补救：直接写 localStorage 把 pending 标为未读
- 新增 useEffect 监听 conversation.pendingDisplayIds，自动重启动画

**Bug ① 一期（P2，1 文件）**：Phase0Identity 加 localStorage checkpoint（仿 Phase1Cards 模式）

**Bug ① 二期（P3，6 文件）**：

- 前端 OnboardingShell mount 时优先调 getOnboardingProgress，降级到 localStorage
- 推荐后续 OnboardingSession 表新增 currentPhase 字段做权威值（需迁移）

### 死循环修复原理
原死循环路径（已切断）：
1. 用户和小夜自然告别 → 后端 auto-ended → 前端 setConversation(null) 但 completedRoles 没加 crush
2. 用户再点小夜 → loadConversation 拿旧 chatId → setConversation(saved) → 不调后端
3. 用户发消息 → 后端 400 "对话已结束" → 前端 setConversation(null) 但 completedRoles 仍没加
4. 死循环

修复后路径：
1. 用户和小夜自然告别 → 后端 auto-ended → 前端 setConversation(null) **且把 crush 加入 completedRoles + removeConversation**
2. 用户再点小夜 → 角色屏显示"已经聊过了"（completedRoles 已包含 crush）
3. 即使误点 → startRoleplay 返回 status='ended' → 前端再次确认 completedRoles + 停留角色屏
4. 不会进入死循环

### 附加保护
- mount 时 listRoleplayChats 同步：即使 localStorage 的 completedRoles 丢失，后端会重新同步
- handleEnd 容错：网络失败不卡死，下次进入让用户重聊

### 改动点
1. **C1: 类型扩展** — RoleplayConversation 新增 pendingDisplayIds 字段
2. **C2: 持久化扩展** — saveConversation 序列化 pendingDisplayIds 为数组；loadConversation 反序列化时还原为 Set，兼容旧数据
3. **C3: pendingMsgIds 合并进 conversation** — 移除独立 state；新增 setPendingDisplayIds 辅助函数（通过 setConversation updater 更新）；约 10 处机械替换
4. **C4: handleBackToRoles 不清 pending** — 让 pendingDisplayIds 随 conversation 持久化；未读数改累加（prev + pending.size，原代码用 = 覆盖）
5. **C5: unmount 补救** — 组件卸载 useEffect 中：如果 conversation 有 pendingDisplayIds，直接写 localStorage 累加未读数 + 确保 conversation 已持久化（含 pendingDisplayIds）
6. **C6: 重启动画 useEffect** — 监听 conversation.pendingDisplayIds：非空且 timers 为空时自动启动 setTimeout 动画逐个移除，让 TypingIndicator 显示

### 设计决策
- **setPendingDisplayIds 辅助函数**：封装 setConversation updater 模式，让 10 处替换更简洁
- **conversation 为 null 时 setPendingDisplayIds 无操作**：updater return prev 直接返回 null，符合预期
- **setConversation(null) 自动清空 pendingDisplayIds**：无需单独清，减少代码重复
- **C6 useEffect 依赖 conversation?.pendingDisplayIds**：Set 引用变化才触发（每次更新都是新 Set 实例）
- **unmount 时直接写 localStorage 而非 setState**：React 反模式但必要（unmount 后 setState 无效）

### 验证
- 前端 `npm --prefix Echo run lint` 退出码 0（3 个预先存在的 poster 错误与本次无关）
- 后端无改动

### 消息一次性全发修复原理
原 bug 根因（与 localStorage 无关）：
- handleSend 中 setConversation updater 一次性把所有 assistantMsgs 加入 messages
- 靠独立 state pendingMsgIds 控制可见性 + setTimeout 逐个移除
- handleBackToRoles 清空 pendingMsgIds 但不清 messages → 跨组件生命周期 localStorage 放大问题

修复后：
- pendingDisplayIds 与 messages 绑定在同一个 conversation 对象上
- handleBackToRoles 不清空 pendingDisplayIds，让它随 conversation 持久化
- 下次进入 loadConversation 恢复 pendingDisplayIds，C6 useEffect 自动重启动画
- 所有消息仍然逐条动画展示，不再一次性全发

### 气泡数字错误修复原理
原 bug：handleBackToRoles 用 pendingMsgIds.size 覆盖（非累加）unreadCount；handleExit 触发 unmount 时不更新 unreadCount
修复后：
- handleBackToRoles 改累加（prev + pending.size）
- unmount 补救：直接写 localStorage 累加未读数
