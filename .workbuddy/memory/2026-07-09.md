- 用户反馈：之前设计的直排海报布局"太呆了"，需要更有创意的结构。
- 提供了 5 个创意方向给生图工具 prompt：声波涟漪图（Echo Ripple）、星座连线图（Constellation Canvas）、双面镜裂像（Split Mirror）、胶片记忆带（Film Strip Diary）、日式杂志封面（Editorial Zine）。每个配了完整 Midjourney/DALL·E prompt。

- 用户问：手机页面上怎么一键生成海报？
- 产出：
  - `Echo/src/utils/poster/PosterCanvas.tsx` — 隐藏海报模板组件（1080×1920 固定尺寸，用 px 绝对布局，与响应式 UI 分离）
  - `Echo/src/utils/poster/usePosterGenerator.ts` — 生成 hook（html-to-image 截图 + Web Share API 保存 + WebView 原生桥接降级）
  - `Echo/src/utils/poster/PosterPage.tsx` — 完整使用示例（页面展示 → 点击生成 → 预览遮罩 → 保存/分享）
  - `outputs/poster-generation-guide.md` — 集成指南（架构图、关键决策、兼容性、后续优化）
- 核心设计：页面 UI 和海报模板是两个独立 DOM 节点，前者响应式布局（375px 宽），后者固定 1080px 宽且隐藏（left:-9999px），截图时临时移入视口。
- 依赖：需 `npm install html-to-image`

## 通知系统改造：评论被回复（B站风格卡片）

- 需求：通知"自己的评论被别人回复了"，参考 B站通知格式（评论者名/动作/新评论内容/时间/回复+删除该通知按钮/灰色引用块显示原评论）。
- 现状诊断：`reply` 类型已用于"回复评论"通知，但格式简陋（仅 title+body 一行文案），无引用上下文，无删除能力，且 `reply` 混用两种语义（直接回复 vs 在你评论过的帖子下评论）。
- 方案：新增专用 `comment_reply` 枚举值 + `notifications.metadata_json` JSONB 列携带结构化上下文，前端用 B站风格卡片渲染。

### 数据库（2 个迁移，避免 ALTER TYPE ADD VALUE 事务问题）
- `20260710000000_add_comment_reply_type` — `ALTER TYPE "NotificationType" ADD VALUE 'comment_reply'`
- `20260710000001_add_notification_metadata_json` — `ALTER TABLE "notifications" ADD COLUMN "metadata_json" JSONB`
- `schema.prisma`：enum NotificationType 加 `comment_reply`；model Notification 加 `metadataJson Json? @map("metadata_json")`

### 后端
- `notification.service.ts`：`create()` 加 `metadata?: Record<string,unknown>` 写入 `metadataJson`；`list()` 返回 `metadata`；新增 `delete(userId,id)`（deleteMany 校验归属）。
- `notification.controller.ts`：新增 `@Delete(':id')` 路由。
- `posts.controller.ts` `notifyCommentRelatedUsers`：opts 加 `commentContent`（原始内容，用于 120 字截断）；回复评论分支中，被回复评论作者的通知改为 `comment_reply` 类型，`title`=评论者名、`body`="回复了我的评论"，metadata 含 `{actorName, replyContent, quotedComment:{content}, postId, commentId, parentCommentId}`。查询 parentComment 时多 select `content`。帖子作者仍收 comment 通知，之前评论者仍收 reply 通知（向后兼容）。

### 前端
- `api/notification.ts`：`NotificationItem.type` 加 `comment_reply`；新增 `NotificationMetadata`/`NotificationQuotedComment` 类型 + `metadata?` 字段；新增 `deleteNotification(id)`（apiDeleteJson）。
- `NotificationBell.tsx`：import `useNavigate` + `deleteNotification` + `Trash2`/`MessageSquare` 图标；TYPE_ICON 加 `comment_reply`；`comment_reply && metadata` 走 B站卡片分支（评论者名+动作/新评论内容/时间/回复按钮+删除该通知/灰色引用块）；`handleDelete` 调 API+更新列表+未读数-1；`handleReplyJump` markRead+关面板+navigate `/post/:postId`；所有通知追加轻量删除图标按钮。

- 前端 `tsc --noEmit`：仅 3 个 **pre-existing** poster 错误（`html-to-image` 未安装 + PosterCanvas/usePosterGenerator 模块缺失），与本次通知改动无关，通知相关代码 0 错误。

## 评论点赞体系（CommentLike 表 + 接口 + UI）

- 需求：上一轮通知卡片缺"点赞"按钮，本轮补齐评论点赞的表与接口。
- 现状：`Like` 表仅支持 Post 点赞且无 HTTP 端点；评论无点赞能力。

### 数据库
- 新增 `CommentLike` model（`comment_likes` 表，复合主键 `commentId+cloneId` 幂等防重，`cloneId` 索引，双外键级联删除）；迁移 `20260710000002_add_comment_likes`。
- `schema.prisma`：`Comment` 加 `likes CommentLike[]`，`DigitalClone` 加 `commentLikes CommentLike[]`。

### 后端
- 新建 `posts/comments.controller.ts`（@Controller('comments')，JwtAuthGuard + 限流 30/min）：
  - `POST /comments/:commentId/like` — 点赞（upsert 幂等）
  - `DELETE /comments/:commentId/like` — 取消（deleteMany 幂等）
  - 均返回 `{ liked, likes }`（count + 当前 clone 是否已赞）
- `posts.module.ts` controllers 注册 `CommentsController`（无需改 app.module）。
- `feed.service.ts` `getOne`：查当前 clone id；评论 include 加 `_count.likes` + `likes:{where:{cloneId},take:1}`；`comments_list` map 顶层与 replies 均返回 `likes`(数字) + `liked`(布尔)。

### 前端
- `api/feed.ts`：`CommentItem` 加 `likes:number`+`liked:boolean`；`loadPostDetail` parseComments 解析；新增 `toggleCommentLike(commentId, liked)`（liked=true→DELETE，false→POST，返回 `{liked,likes}` 或 null）。
- `PostDetailView.tsx`：import `Heart`+`toggleCommentLike`；`applyCommentLike` 递归更新评论树；`handleToggleLike` 乐观更新→接口→校正/回滚；顶层评论与子评论在"回复"按钮后加点赞按钮（Heart 图标，已赞蓝色 fill-current+数字，未赞灰色描边）。
