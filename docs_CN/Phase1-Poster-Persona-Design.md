# Echo Phase 1 入驻海报宣传页设计方案

## 设计文档 | 版本：1.0 | 状态：提案

---

## 1. 背景与目标

### 1.1 为什么做海报页

Echo Phase 1 的入驻问卷已经设计成一套高信息密度的「自我探索」体验：18 张情境卡片、四阶段对话式采集、最终生成人格画像（Persona Sketch）和理想伴侣画像（Ideal Partner Sketch）。如果用户只完成一次就离开，这套投入的边际价值会被浪费。

一个类似 MBTI/SBTI 风格的**入驻结果海报页**可以达成两个目标：

- **体验价值**：把抽象维度分数转化为可感知的、可分享的视觉身份，让用户觉得「我被读懂了」。
- **传播价值**：通过个人关键字 + 理想型关键字的展示，制造社交货币，鼓励转发拉新。

### 1.2 与 MBTI 的本质区别

Echo 的测量体系是**连续维度**（Big Five、MFT、依恋四象限、时间观等），不是 MBTI 式的 16 型分类。因此海报页需要：

- 用「形象原型」增加记忆点，但**不强加固定类型**；
- 用「关键字云」保留维度的连续性和丰富度；
- 在页面中说明：「这是你的回声精灵，而不是对你的定义。」

## 2. 核心设计原则

| 原则 | 说明 |
|------|------|
| **温暖科技** | 不冰冷、不机械，用圆润角色和柔和渐变传达 Echo 的品牌调性。 |
| **人格化** | 每个用户得到的是一个独特的「回声精灵」形象，而非分数报表。 |
| **可分享** | 9:16 海报比例，适合朋友圈、小红书、微博。 |
| **双向表达** | 同时展示「我是谁」和「我在找谁」，呼应理想型互匹配机制。 |
| **成长感** | 暗示用户可以继续训练分身，海报只是起点。 |

## 3. 12 个回声精灵形象原型

基于 Phase 1 采集的维度，我们设计 12 个可选的「回声精灵」原型。这些原型不是人格类型，而是**性格气质的可视化聚合**。

### 3.1 原型设计维度

- **Big Five 主导维度**：E（外向）/ O（开放）/ C（尽责）/ N（神经质）/ A（宜人性）
- **依恋风格**：secure / preoccupied / dismissing / fearful
- **时间观**：future / past / present / balanced
- **核心情绪基调**：温暖 / 探索 / 稳定 / 敏感 / 独立 / 守护

### 3.3 原型映射规则（算法侧）

映射不采用硬分类，而是**加权打分 + 置信度**。

```typescript
// 伪代码
function selectElfArchetype(dimensions: DimensionScores): ElfArchetypeResult {
  const candidates: Record<ElfId, number> = {};

  for (const elf of ELVES) {
    let score = 0;
    for (const rule of elf.rules) {
      const value = dimensions[rule.dimension];
      score += rule.weight * (rule.direction === 'high' ? value : -value);
    }
    candidates[elf.id] = score;
  }

  const sorted = Object.entries(candidates).sort((a, b) => b[1] - a[1]);
  const [topId, topScore] = sorted[0];
  const runnerUpScore = sorted[1][1];
  const confidence = (topScore - runnerUpScore) / (Math.abs(topScore) + 1); // 归一化差距

  return {
    elfId: topId,
    confidence: clamp(confidence, 0, 1),
    topTwo: sorted.slice(0, 2).map(([id]) => id),
  };
}
```

**示例规则**：栖光鹿（ELF-01）

```typescript
{
  id: 'ELF-01',
  name: '栖光鹿',
  rules: [
    { dimension: 'agreeableness', weight: 1.2, direction: 'high' },
    { dimension: 'neuroticism', weight: 1.0, direction: 'low' },
    { dimension: 'attachmentAnxiety', weight: 0.8, direction: 'low' },
    { dimension: 'attachmentAvoidance', weight: 0.8, direction: 'low' },
    { dimension: 'timePresent', weight: 0.6, direction: 'high' },
  ],
}
```

**置信度处理**：

- `confidence >= 0.3`：展示单一原型，标题为「你的回声精灵是栖光鹿」。
- `confidence < 0.3`：展示「双原型融合」，例如「栖光鹿 + 探梦狐的混合体」，并说明两个气质都在你身上出现。

## 4. 个人关键字与理想型关键字

### 4.1 个人关键字来源

个人关键字从以下数据中抽取，每类最多 2 个，最终展示 3-5 个：

| 来源 | 抽取规则 | 示例 |
|------|----------|------|
| **Big Five 主导** | 取绝对值最高的两个维度，按方向给词 | 高 E → 外向；低 E → 内敛；高 O → 开放；低 O → 务实 |
| **依恋风格** | 直接映射为关键词 | 安全型 → 稳定；痴迷型 → 依恋；疏离型 → 独立；恐惧型 → 复杂 |
| **时间观** | 取最高分项 | 未来导向 → 前瞻；过去导向 → 怀旧；当下导向 → 活在当下 |
| **MFT 道德基础** | 取最高两项 | 高 care → 共情；高 fairness → 公正；高 authority → 秩序；高 loyalty → 忠诚 |
| **用户自选语气标签** | 来自 M2 语言指纹 | 松弛、直接、温柔、幽默、理性、热情 |
| **自由文本高频词** | 从 Persona Sketch 提取 | 勇敢、自由、敏感、真诚 |

**示例关键字**：

> 温暖 · 倾听 · 稳定 · 活在当下 · 真诚

### 4.2 理想型关键字来源

理想型关键字从 Ideal Partner Sketch 的 4 个维度 + 用户自由文本中提取：

| 维度 | 高分关键词 | 低分关键词 |
|------|------------|------------|
| `needEmotionalSafety` | 情感确认、安全感、被回应 | 独立空间、情绪自给 |
| `needSpaceRespect` | 边界感、独处、个人空间 | 陪伴、黏在一起 |
| `needDirectCommunication` | 直球、有话直说 | 委婉、默契、心照不宣 |
| `needConflictResolution` | 当场解决、不隔夜 | 冷静消化、各自沉淀 |

**示例理想型关键字**：

> 情感确认 · 边界感 · 有话直说 · 不隔夜

### 4.3 关键字的「去标签化」文案

海报页需要在展示关键字的同时，避免让用户觉得被贴了标签。建议在每个关键字模块底部加一行小字：

> "这些词是你此刻的回声，不是定义。"

### 5.3 各区详细设计

#### 顶部区（高度约 120px）

- 左侧：Echo Logo（48px 高）
- 右侧：小字「Phase 1 入驻结果」
- 背景：半透明毛玻璃胶囊，不抢主视觉

#### 主视觉区（高度约 520px）

- 精灵插画：居中，占画面宽度 70%-80%
- 插画下方：
  - 原型名：「栖光鹿」—— 32px，粗体，品牌主色
  - 原型代号：「ELF-01」—— 14px，60% 透明度
- 插画周围：少量漂浮小元素（心形、星星、声波线），增强灵动感

#### 身份宣言区（高度约 180px）

- 文案：
  - 小标题：「你的回声」（14px，灰色）
  - 主文案：精灵宣言（24px，深色，居中）
- 使用引号装饰，引号颜色为精灵主色 30% 透明度

#### 个人关键字区（高度约 200px）

- 标题：「我的关键字」（16px，深色）
- 标签：胶囊状，背景为精灵主色 15% 透明度，文字为精灵主色 100%
- 标签间距：12px
- 标签尺寸：字号 14px，padding 8px 16px

#### 理想型关键字区（高度约 200px）

- 标题：「我想遇见」（16px，深色）
- 标签：胶囊状，背景为互补色 15% 透明度，文字为互补色 100%
  - 例如栖光鹿主色为珊瑚橙，理想型可用深海蓝或柔光紫作为互补色
- 两个区域之间用细线或分隔符区分

#### 可视化区（可选，高度约 260px）

两种方案：

**方案 A：雷达图**
- 展示用户的 Big Five 五维或理想型四维
- 使用浅色填充 + 主色描边
- 标题：「我的回声波形」

**方案 B：双精灵剪影**
- 左侧是用户的回声精灵，右侧是「适合我的回声精灵」的抽象剪影
- 中间用声波线连接
- 呼应双向互匹配机制

#### CTA 区（高度约 240px）

- 主按钮：「生成我的分身」
  - 背景：精灵主色渐变
  - 文字：白色，18px，粗体
  - 圆角：999px（胶囊按钮）
  - 阴影：主色 20% 透明度，Y 偏移 8px
- 次按钮：「分享给朋友」
  - 描边按钮，文字为主色
- 二维码：右下角，带 slogan
  - slogan：「测测你的回声精灵」

### 5.4 响应式与导出

- 海报页在 App 内以 9:16 展示。
- 提供「保存到相册」和「直接分享」两个出口。
- 导出图片：PNG，1080 × 1920，压缩后 ≤ 800KB。

## 6. 分享与传播机制

### 6.1 分享路径

| 路径 | 体验 |
|------|------|
| **海报分享** | 用户保存图片，分享到朋友圈/小红书/微博。 |
| **链接分享** | 生成带邀请码的短链接，接收方打开后进入「测测你的回声精灵」落地页。 |
| **小程序卡片** | 若未来接入小程序，可用精灵形象生成卡片。 |

### 6.2 落地页设计（接收方视角）

接收方点击分享链接后，进入简化版入驻：

1. 展示「朋友的小鹿形象」作为引子。
2. 文案：「想知道你的回声精灵是什么吗？6 分钟测一测。」
3. 直接进入 Phase 1 入驻流程。
4. 完成后也生成自己的海报，形成二次传播。

### 6.3 邀请激励（可选）

- 分享后可获得「回声币」或解锁更多精灵皮肤。
- 但避免过度游戏化，保持温暖调性。

### 7.2 推荐新增端点

```typescript
// GET /onboarding/poster
interface OnboardingPosterResponse {
  elfId: string;              // ELF-01
  elfName: string;            // 栖光鹿
  colorPrimary: string;       // #FF6B6B
  colorSecondary: string;     // 互补色
  illustrationPrompt: string;   // 用于 AI 生成插画的 prompt

  idealKeywords: string[];      // 理想型关键字

  secondaryElfId?: string;      // 次原型，低置信度时返回
  radarData: { label: string; value: number; }[]; // 雷达图数据
}
```

### 7.3 插画生成方案

**方案 A：预置插画库（推荐 Phase 1）**
- 12 个原型各准备 1 张高质量插画（PNG/SVG）。
- 优点：加载快、风格统一、成本低。
- 缺点：扩展新原型需要重新设计。

**方案 B：AI 实时生成**
- 通过后端或 MCP 图像生成工具，根据 `illustrationPrompt` 生成。
- 优点：千人千面、可随 Persona Sketch 变化。
- 缺点：生成耗时、风格一致性难控。

**建议**：Phase 1 用方案 A，预置 12 张插画 + 基于主色调的动态背景；后续用方案 B 做高级版或付费版。

## 8. 迭代与测试计划

### 8.1 第一阶段：MVP 海报

- 12 个原型 + 关键字云 + 基础海报 UI
- 预置插画
- 导出 PNG

- 根据 Persona Sketch 微调宣言文案

- 邀请落地页
- 分享激励

### 8.4 测试指标

- 海报保存率
- 分享率
- 落地页转化率（海报 → 新用户入驻）
- 用户「这就是我」的认同感（可用问卷或 NPS）
