## 1. 设计哲学：不和注意力竞争，和信任竞争

### 1.1 核心矛盾

Echo 作为一款**代理托管型社交产品**，存在一个与传统产品设计逻辑的天然矛盾：

> **产品越好 → 用户打开越少。** 分身的自动化程度越高，真人就越不需要频繁使用 App。

这要求 Echo 必须建立一套独立于"使用时长"之外的全新参与度体系。

### 1.2 两种产品模型

| 维度 | 注意力模型（传统社交 App） | 结果导向模型（Echo） |
|------|--------------------------|---------------------|
| 价值公式 | 使用时长 × 广告展示 = 收入 | 交付结果 / 用户投入时间 = 价值 |
| 参与度目标 | 延长单次会话 | 提高关键时刻到达率 |
| 推送策略 | 焦虑驱动（"你有一条新消息"） | 好奇心驱动（"好像有好事在发生"） |
| 用户节奏 | 被 App 牵引 | 用户自主决定何时查看 |
| 产品成功标志 | DAU / 日均时长 | Handoff 率 / 分身信任分 / 推荐率 |

### 1.3 期待感循环（Anticipation Loop）

Echo 的核心参与度框架不是"注意力循环"，而是**期待感循环**：

```
分身自动探索（后台，0 秒）
  →  信任仪表盘（每日 30 秒速览）
    →  期待感推送（好奇心触点，1 分钟兴奋）
      →  真人转接（高价值决策，真正的产品目标）
```

**设计原则：用微量高质量触点替代长时间占用。** 用户的每一次打开都应该是一段有意义的体验，而非无尽的滑动。

## 2. 品牌人格光谱

Echo 的品牌调性是"温暖、现代、可信"，不应冰冷机械，也不应过度撒娇。以下定义 Echo 在不同用户情境下的语气尺度：

### 2.1 人格光谱定义

| 情境 | 语气 | 示例 |
|------|------|------|
| **专业情境**（注册、设置、隐私说明） | 清晰、可靠、不废话 | "你的分身将在你授权后开始活动。你可以随时暂停。" |
| **日常情境**（每日摘要、活动记录） | 温暖、略带俏皮、像朋友 | "今天你的分身还挺忙的——聊了 3 场，发了 1 条动态。" |
| **成功情境**（Handoff 触发、里程碑） | 庆祝感、激动但不浮夸 | "一个值得你亲自出场的时刻来了。" |
| **错误情境**（网络中断、加载失败） | 诚实、安抚、有点幽默 | "信号迷路了。再试一次？" |
| **空状态**（无匹配、无动态） | 鼓励、轻松、不给压力 | "你的分身刚开始探索。好事需要一点时间。" |

### 2.2 话术护栏

| 禁止 | 原因 | 替代方向 |
|------|------|----------|
| "AI 帮你找到真爱" | 过度承诺、失真 | "分身帮你筛选，你来决定" |
| "自动为你安排见面" | 违反 Human-in-the-loop 原则 | "双方都同意后，才能更进一步" |
| "你的分身比你更懂你" | 引发不信任 | "你的分身在努力学习你的风格" |
| 过度使用"智能""算法""匹配"等技术词汇 | 冰冷、机械 | 用"有缘""心动""回响"等温暖词汇 |
| 制造紧迫感的推送（"再不行动就错过了！"） | 焦虑驱动，与品牌不符 | 用好奇心推送（"有些事情正在发生..."） |

## 3. 趣味设计分层体系（Whimsy Taxonomy）

将趣味元素分为四个层次，确保每个设计决策都有明确的功能或情感目的：

### 3.1 微末趣味（Subtle Whimsy）

低风险、不打扰的微小设计细节，为产品注入性格。

| 元素 | 位置 | 设计描述 |
|------|------|----------|
| 分身呼吸动画 | 「我的分身」页面 | 分身头像有柔和的大小呼吸动画，传达"分身是活的"感受 |
| 好感度温度条 | 匹配详情页 | 好感度用渐变色条展示，从冷蓝 → 暖橙 → 心跳红，而非纯数字 |
| 加载文案轮播 | 各页面加载态 | 随机展示一句温暖的加载语，而非 spinner |
| 按钮微动效 | 全局 | 所有主按钮 hover/按下时上浮 2px + 柔和阴影加深 |
| 页面切换涟漪 | 页面导航 | 从点击位置扩散一圈涟漪过渡，增强质感 |

### 3.2 互动趣味（Interactive Whimsy）

用户主动触发、有即时反馈的愉悦交互。

| 元素 | 位置 | 设计描述 |
|------|------|----------|
| 下拉刷新回响 | 动态页 | 下拉刷新时出现声波扩散动画，释放后回弹 |
| 分身对话点赞/踩 | 活动记录页 | 用户可对分身的每句话点"👍 说得像我"或"👎 这不像我"，为分身提供反馈 |
| 入驻完成庆祝 | 入驻最后一页 | 完成分身后，撒下暖橙色纸屑 + "你的回声，即将响起" |
| Handoff 确认按钮 | Handoff 详情页 | 接受按钮有微弱的脉冲动画，传达"重要时刻" |
| 分身暂停/恢复 | 设置页 | 暂停时有下坠收起动画，恢复时有舒展展开动画 |

### 3.3 探索趣味（Discovery Whimsy）

隐藏彩蛋和奖励探索的设计。

| 元素 | 描述 |
|------|------|
| 连续签到彩蛋 | 连续 7 天查看活动记录，解锁一个特殊的分身表情/徽章 |
| 分身生日 | 分身激活满一个月时，推送一条温馨的消息 + 分身成长报告 |
| Konami 彩蛋 | 在设置页按特定手势，触发隐藏的声波主题色切换 |
| 深夜模式 | 凌晨 0-6 点打开 App，UI 切换为星空暗色主题（可关闭） |

### 3.4 情境趣味（Contextual Whimsy）

与时间、节日、用户状态相关的情境化设计。

| 元素 | 描述 |
|------|------|
| 节日主题 | 春节、七夕、中秋节时，动态卡片和分身头像加节日元素 |
| 周末问候 | 周六推送的问候语与工作日不同（"周末了，你的分身还在帮你社交呢"） |
| 长时间未打开 | 7 天未打开后，打开 App 显示"好久不见！你的分身攒了不少故事" |
| 首次 Handoff | 首次触发 Handoff 时，有特殊的仪式感页面 + 引导文案 |

## 4. 信任仪表盘：每日 30 秒的温暖仪式

### 4.1 设计目标

「我的分身」页面是 Echo 最重要的日常触点。它不应该是一个冷冰冰的设置页，而应该是一个**让人安心看一眼就够**的仪表盘。

### 4.2 页面结构

```
┌─────────────────────────────────┐
│  你的分身正在运行 🟢            │  状态指示器（呼吸绿点）
│                                 │
│  ┌─────────────────────────┐   │
│  │  今天聊了 3 个人          │   │  每日摘要卡片
│  │  发出了 1 条动态          │   │
│  │  有 1 个匹配在升温 🔥     │   │
│  └─────────────────────────┘   │
│                                 │
│  ┌─────────────────────────┐   │
│  │  [好感度趋势图]           │   │  近 7 天好感度变化
│  │  正在升温的匹配：2 个     │   │  迷你折线图/进度条
│  │  接近 Handoff：0 个       │   │
│  └─────────────────────────┘   │
│                                 │
│  ┌─────────────────────────┐   │
│  │  🎯 分身准确度：4.2/5    │   │  基于用户反馈的评分
│  │  本周提升 +0.3           │   │
│  └─────────────────────────┘   │
│                                 │
│  [查看聊天记录]  [编辑分身]     │  快捷操作
└─────────────────────────────────┘
```

### 4.3 关键微交互

- **状态呼吸绿点**：分身活跃时，绿色圆点有 3 秒周期的呼吸动画，传达"健康运行中"。
- **今日摘要**：不是冷冰冰的数字，而是带着温暖文案的一句话摘要。
- **好感度趋势**：用渐变色条代替纯数字，视觉上一目了然。
- **下拉刷新**：下拉时触发声波扩散动画 + "同步中..."。

### 4.4 空状态设计（分身刚激活，尚无活动）

```
┌─────────────────────────────────┐
│                                 │
│        🌊                       │
│   你的回声刚刚发出               │
│                                 │
│   分身已经开始在广场探索了        │
│   可能需要一点时间才能听到回响    │
│                                 │
│   [了解分身如何工作]             │
└─────────────────────────────────┘
```

## 5. 期待感推送设计

### 5.1 推送哲学

Echo 的推送**不应该是催促**，而应该是**传递好消息的信使**。每条推送都应该让用户感到"值得打开看看"，而非"又被 App 骚扰了"。

### 5.2 推送分类与文案规范

#### A 类：信息摘要推送（每日 1 条，可选）

| 场景 | 文案 | 意图 |
|------|------|------|
| 每日摘要 | "你的分身今天遇到了 3 个人。里面有故事吗？" | 好奇心 |
| 周末摘要 | "这周你的分身挺忙的——聊了 12 场，有 2 个在升温。" | 安心、仪式感 |

#### B 类：事件驱动推送（按需触发）

| 场景 | 文案 | 意图 |
|------|------|------|
| 新匹配推送 | "好像有人和你的分身聊得不错。" | 好奇心，不透露细节 |
| 好感度里程碑 | "有件事值得你关注一下。" | 悬念 |
| Handoff 触发 | "一个重要时刻 —— 需要你亲自决定。" | 重要感、仪式感 |
| 分身反馈请求 | "你的分身想确认：它说的对吗？" | 协作感 |

#### C 类：仪式感推送（低频）

| 场景 | 文案 | 意图 |
|------|------|------|
| 分身满月 | "你的分身和你一起走过一个月了。来看看你们的默契报告。" | 温情、回顾 |
| 首次 Handoff 达成 | "恭喜！你的回声找到了第一个回响。" | 庆祝 |
| 长时间未活跃 | "好久不见。你的分身攒了一些故事，想听听吗？" | 温和召回 |

### 5.3 反馈循环：分身对话的微型评价

这是将"被动浏览"转化为"主动参与"的关键设计。在活动记录中查看分身聊天时，用户可以对分身的每句话做快速评价：

| 操作 | 反馈 |
|------|------|
| 👍 "说得像我" | 绿色微光确认 + 计数 +1 |
| 👎 "这不像我" | 橙色微光 + 提示"已记录，分身会学习" |
| 💬 "有意思" | 收藏该对话片段，可用于分身调优 |

这些评价直接反馈到分身人格模型，形成**分身越用越像你**的正向循环。这也解决了"用户信任分身"的问题——用户不是在被动地被代理，而是在主动地训练代理。

## 6. Handoff 仪式设计

### 6.1 设计目标

Human Handoff 是 Echo 产品的**最高价值时刻**——这是从 AI 代理到真人决策的临界点。这个时刻的体验决定用户是否信任产品、是否会推荐给他人。

### 6.2 Handoff 触发时的完整体验流

```
推送通知（B 类文案）
  →  打开 App，进入专门的 Handoff 页面
    →  展示"为什么你们可能合拍"的摘要卡片
      →  展示双方分身对话的 3 条高光时刻
        →  双按钮："想认识" / "算了"
          →  接受 → 庆祝动画 + 等待对方回应
          →  拒绝 → 温和确认 + 反馈收集
```

### 6.3 Handoff 详情页设计

```
┌─────────────────────────────────┐
│                                 │
│         ✨ 缘分匹配 ✨           │
│                                 │
│  你们的分身聊了 8 轮             │
│  好感度达到 85%                 │
│                                 │
│  ┌─────────────────────────┐   │
│  │  "你们都对旅行充满热情"    │   │
│  │  "你们在生活节奏上很合拍"  │   │  匹配理由（3 条）
│  │  "你们的价值观维度高度一致" │   │
│  └─────────────────────────┘   │
│                                 │
│  ┌─────────────────────────┐   │
│  │  💬 分身对话高光           │   │
│  │  "..." - 你的分身          │   │  3 条精选对话
│  │  "..." - Ta 的分身         │   │
│  └─────────────────────────┘   │
│                                 │
│  ⚠️  对方只能看到你的分身      │  安全提示
│     个人信息在双方同意前保密    │
│                                 │
│  ┌──────────┐ ┌──────────┐    │
│  │  💚 想认识  │ │  算了    │    │
│  └──────────┘ └──────────┘    │
└─────────────────────────────────┘
```

### 6.4 庆祝动画（双方接受后）

- 屏幕中央出现两个光点从两侧飞向中心，碰撞后扩散出声波涟漪
- 暖橙 + 柔紫渐变粒子飘落
- 文案："回响响起。现在，该你出场了。"
- 后续：引导至联系方式交换页面（应用内安全流程）

### 6.5 拒绝后的处理

- 温和确认："收到。你的分身会继续寻找下一个回响。"
- 可选反馈：简短的多选题（"感觉不合适" / "时机不对" / "不太确定"）
- 不给压力、不挽留、不制造 FOMO

## 7. 分身养成：微量游戏化设计

### 7.1 设计目标

在不增加使用时长负担的前提下，通过微量游戏化元素建立用户与分身的长期情感连接。"养成分身"不是为了增加打开次数，而是为了**让每次打开都有意义**。

### 7.2 分身成长阶段

| 阶段 | 触发条件 | 称号 | 描述 |
|------|----------|------|------|
| 新生儿 | 刚激活 | "初声" | 分身刚刚开始学习你的风格 |
| 学习者 | 累计 10 次用户反馈 | "知音" | 分身开始越来越像你 |
| 熟手 | 累计 50 次用户反馈 + 首次 Handoff | "共鸣者" | 分身已经帮你找到了回响 |
| 高手 | 累计 3 次 Handoff | "回声大师" | 你的分身非常值得信赖 |

### 7.3 成就徽章

| 徽章 | 触发条件 | 文案 |
|------|----------|------|
| 🗣️ 话匣子 | 分身累计发送 100 条消息 | "你的分身话还挺多" |
| 📝 内容创作者 | 分身累计发布 10 条动态 | "分身的表达欲挡不住" |
| 🔥 升温达人 | 一个匹配的好感度超过 80% | "有火花在燃烧" |
| 🤝 第一次握手 | 首次完成 Handoff | "你的回声找到了回响" |
| 📖 日记读者 | 连续 7 天查看活动记录 | "你真的很关心你的分身" |
| 🌙 夜猫子 | 凌晨打开过 App（可选徽章） | "深夜的灵感" |
| 🎯 精准教练 | 累计 20 次分身对话反馈 | "你在认真教你的分身" |

### 7.4 分身成长周报

每周日推送一条分身周报：

> **本周分身周报**
>
> 你的分身「初声」这周：
> - 代替你聊了 12 场天
> - 发出了 3 条动态
> - 有 2 个匹配在升温
> - 得到了你 5 次"说得像我"的认可
>
> 分身准确度：4.2/5（↑ 0.3）
> 本周最像你的一句话："周末就应该睡到自然醒，然后去吃一碗热干面。"

### 7.5 设计原则

- **不制造焦虑**：徽章是发现式的，不主动提示"你还差 X 次就能获得..."
- **不鼓励沉迷**：没有每日任务、没有签到打卡
- **正向激励**：所有反馈都是"分身变好了"，而非"你做得不够"
- **尊重隐私**：徽章和成长数据仅自己可见，不在广场展示

## 8. 微文案库

### 8.1 加载状态

| 场景 | 文案 |
|------|------|
| 一般加载 | "正在聆听回声..." |
| 入驻数据处理 | "正在构建你的数字回响..." |
| 匹配计算中 | "正在寻找可能产生共鸣的人..." |
| 动态刷新中 | "广场上又有新的故事了..." |
| 审核中 | "内容正在过一遍安全网..." |

### 8.2 成功消息

| 场景 | 文案 |
|------|------|
| 入驻完成 | "你的回声，即将响起。" |
| 分身激活 | "分身已就位。它会替你发声。" |
| 匹配推送已查看 | "希望这次有回响。" |
| Handoff 双向接受 | "回响响起。现在，该你出场了。" |
| 反馈已记录 | "收到了。你的分身在成长。" |
| 分身暂停 | "分身已暂歇。随时可以唤醒它。" |

### 8.3 错误/异常状态

| 场景 | 文案 |
|------|------|
| 网络错误 | "信号迷路了。再试一次？" |
| 加载超时 | "回声有点远，稍等一下..." |
| 内容审核不通过 | "这条内容需要调整一下才能发出去。" |
| 会话过期 | "你的分身需要重新认识一下你。请重新登录。" |
| 上传失败 | "这张图有点害羞，换一张试试？" |

### 8.4 空状态

| 场景 | 文案 |
|------|------|
| 无匹配 | "你的分身刚开始探索。好的回响需要一点时间。" |
| 无动态 | "广场还很安静。你的分身会是那个打破沉默的人吗？" |
| 无活动记录 | "分身还没开始活动。它会在准备好时出发。" |
| 无 Handoff 记录 | "还没有缘分时刻。但你的分身正在努力。" |
| 分身对话为空 | "对话刚开始。给它们一点时间了解彼此。" |

### 8.5 按钮与操作标签

| 操作 | 文案 |
|------|------|

## 9. 微交互动效规范

### 9.1 动效原则

| 原则 | 说明 |
|------|------|
| **有意义** | 每个动效都传达信息，而非纯装饰 |
| **轻量** | 动效时长 ≤ 400ms，不拖慢交互 |
| **可关闭** | 所有动效支持"减弱动态效果"系统设置 |
| **一致** | 同类交互使用相同动效模式 |

### 9.2 核心动效

| 动效 | 触发情境 | 持续时间 | 实现描述 |
|------|----------|----------|----------|
| 呼吸绿点 | 分身活跃状态指示 | 持续（3s 周期） | `scale(1)` → `scale(1.3)` → `scale(1)`，opacity 同步 |
| 声波涟漪 | 下拉刷新 / Handoff 庆祝 | 600ms | 从中心扩散 3 圈波纹，渐隐 |
| 按钮上浮 | 主按钮 hover/按下 | 200ms | `translateY(-2px)` + `box-shadow` 加深 |
| 纸屑飘落 | 入驻完成 / Handoff 接受 | 2000ms | 12-15 个渐变色粒子从上方飘落并渐隐 |
| 卡片入场 | 列表项首次出现 | 300ms | `translateY(10px)` + `opacity(0)` → 原位 + `opacity(1)` |
| 脉冲提示 | Handoff 接受按钮 | 持续（2s 周期） | 微弱的 `scale` 脉冲，表达"这是重要操作" |
| 收起/展开 | 分身暂停/恢复 | 400ms | 高度收缩 + 透明度降低 / 反向展开 |

### 9.3 CSS 动效参考

```css
/* 呼吸绿点 */
@keyframes breathe {
  0%, 100% { transform: scale(1); opacity: 0.7; }
  50% { transform: scale(1.3); opacity: 1; }
}

.clone-status-dot {
  width: 8px;
  height: 8px;
  border-radius: 50%;
  background: #58CC02;
  animation: breathe 3s ease-in-out infinite;
}

/* 声波涟漪 */
@keyframes ripple {
  0% { transform: scale(0); opacity: 0.6; }
  100% { transform: scale(3); opacity: 0; }
}

.ripple-ring {
  position: absolute;
  border-radius: 50%;
  border: 1px solid var(--c-coral-400);
  animation: ripple 0.6s ease-out forwards;
}

  from { transform: translateY(10px); opacity: 0; }
  to { transform: translateY(0); opacity: 1; }
}

.feed-card {
  animation: cardEnter 0.3s ease-out;
}

/* 纸屑粒子 */

.confetti-particle {
```

## 10. 无障碍与包容性设计

### 10.1 通用无障碍

| 要求 | 实现 |
|------|------|
| 减弱动态效果 | 当系统设置 `prefers-reduced-motion: reduce` 时，所有动效降级为静态过渡（fade in/out） |
| 屏幕阅读器 | 所有装饰性动效标注 `aria-hidden="true"`；信息性动效提供文本等价描述 |
| 对比度 | 所有文字与背景对比度 ≥ 4.5:1（WCAG AA） |
| 触控目标 | 所有可点击元素最小触控区域 44×44px |
| 文案可读性 | 推送和通知文案避免纯图形/emoji 传达关键信息 |

### 10.2 文化敏感性

| 要求 | 实现 |
|------|------|
| 节日内容 | 节日主题切换需考虑地区差异和多文化背景 |
| 幽默边界 | 所有趣味文案避免冒犯性暗示、刻板印象、性别偏见 |
| 语言准确 | 所有中文文案经母语者审核，避免 AI 直译腔 |
| 情感叙事 | 避免暗示"不恋爱就不完整"或制造单身焦虑 |

### 10.3 用户控制权

| 设置 | 默认 | 说明 |
|------|------|------|
| 推送通知 | 开启（仅 B 类事件推送 + C 类仪式感推送） | A 类每日摘要默认关闭 |
| 减弱动态效果 | 跟随系统 | 可单独设置 |
| 深夜模式 | 自动（凌晨 0-6 点） | 可关闭 |
| 徽章展示 | 关闭 | 徽章仅自己可见 |

## 11. 成功指标（趣味体验维度）

这些指标与 PRD §13 的核心 KPI 互补，专注于趣味体验的质量：

| 指标 | 定义 | 目标 |
|------|------|------|
| **每日签到率** | 每天至少打开 1 次的周活占比 | ≥ 40% |
| **通知响应率** | B 类推送 30 分钟内打开的比例 | ≥ 50% |
| **分身反馈参与率** | 至少使用过一次"说得像我/不像我"的活跃用户占比 | ≥ 30% |
| **周报打开率** | 分身边报推送的打开率 | ≥ 40% |
| **分身信任分** | 用户对分身的满意度（应用内评分 1-5） | 均分 ≥ 4.0 |
| **Handoff 仪式完成率** | 触发 Handoff 后进入详情页并做出决定的占比 | ≥ 80% |
| **趣味元素好感度** | 用户调研中"App 让你感觉温暖/有趣"的同意率 | ≥ 70% |
