## 一、当前问卷的问题诊断

### 用一句话说

> 当前问卷在收集"名片信息 + 笔迹样本"来构建一个人——这会得到一个"看起来有点像"的克隆，但经不起任何稍微偏离预设场景的对话。

### 具体问题清单

| 维度 | 当前覆盖 | 缺失 |
|------|---------|------|
| **身份信息** | 昵称、城市、8个预设兴趣标签 | 职业/领域、生活阶段、关键经历、性格自述、日常习惯 |
| **语言风格** | 3个场景选择 + 6个语气标签 + 1条可选自由文本 | 场景太少（无兴奋/安慰/吐槽场景）、无真实语气样本采集机制（缺少对具体刺激的自然回复）、无口头禅/惯用语系统采集 |
| **价值观/观点** | 2道二选一（约会节奏 + 分歧方式），都限于约会场景 | 日常话题的观点判断、信任观、幸福观、矛盾点、用户曾"改变想法"的经历 |
| **关系模式** | 完全缺失 | 用户在不同亲密度下如何切换表达、什么让用户感到被理解/被冒犯、情绪反应模式 |
| **深度对话** | 4-8轮非结构化闲聊 | 对话缺乏针对性追问、无法暴露用户的矛盾与复杂性、样本量不够做语言学分析 |

### 最致命的单一问题

当前系统构建 persona 的 prompt seed（`buildPersonaSeedFromSurvey`）本质上只有 **5 行中文**：

> "Echo 数字分身，城市 XX，目标 XX。兴趣：XX、XX。语气标签：XX。典型回复风格：XX。用户常发的一句：XX。价值观选择：XX。"

用 5 行信息让 LLM 扮演一个复杂的人类——这就是为什么克隆出来的分身"不太像"。

## 二、重构方案：五层人格采集模型

### 核心理念

一个"像你"的数字分身，需要的不是更多数据，而是**覆盖你心智模型的五个维度**：

```
          ┌──────────────────────┐
          │   Module 5: 深度对话   │  ← 整合、揭示矛盾、补盲区
          │  (Integration Layer)  │
          ├──────────────────────┤
          │  Module 4: 关系地图   │  ← 社交适应模式
          │  Module 3: 信念系统   │  ← 判断框架
          │  Module 2: 语言指纹   │  ← 表达方式
          │  Module 1: 身份基座   │  ← 事实知识
          └──────────────────────┘
```

这五个模块分别回答一个问题：

| 模块 | 回答的问题 | 为什么不可缺失 |
|------|-----------|---------------|
| M1 身份基座 | "克隆应该知道什么？" | 没有它，克隆不知道你喜欢什么、经历过什么、日常怎么过 |
| M2 语言指纹 | "克隆应该怎么说？" | 没有它，克隆会"说对话"但不像"你说的话" |
| M3 信念系统 | "克隆应该怎么判断？" | 没有它，克隆在遇到新话题时只能猜或中立 |
| M4 关系地图 | "克隆应该怎么适应？" | 没有它，克隆对所有人说话方式一样——真正的人不会 |
| M5 深度对话 | "哪里还有盲区？" | 没有它，以上四个模块的遗漏不会被发现，矛盾不会被保留 |

### 与当前系统的映射

```
当前问卷                          →  重构后
------------------------------------------
Step 1: 基础画像                   →  M1 身份基座（深度大幅扩展）
Step 2: 语言风格 (3场景)           →  M2 语言指纹（场景扩充 + 回复消息 + 补全句子）
Step 3: 语气标签                   →  M2 语言指纹（保留但加证据）
Step 4: 常用表达 (可选)            →  M2 语言指纹（拆成补全句子 + 回复消息）
Step 5: 价值观 (2道)               →  M3 信念系统（4-5道 + why追问 + 日常话题）
无对应                            →  M4 关系地图（全新）
Step 7: AI对话 (4-8轮)            →  M5 深度对话（导向式追问，8-12轮）
Step 6: 分身授权                   →  保留
Step 8: 孵化分身                   →  保留
```

## 三、各模块详细设计

### Module 1：身份基座（Identity Baseline）

**目标**：让克隆拥有用户的事实知识库——在对话中"知道该说什么"。

**UX 设计原则**：
- 使用进度友好的 UI（每次 2-3 个问题，不要长表单）
- 每个问题都有"跳过"选项
- 混用选择 + 开放文本，降低填写摩擦

| 问题 | 类型 | 为什么重要 |
|------|------|-----------|
| **1.1 昵称** | 文本输入 | 保留原有，克隆需要称呼自己 |
| **1.2 城市** | 文本/选择 | 保留原有，影响匹配和语境 |
| **1.3 职业/领域** | 选择+文本 | 全新：决定克隆在话题中能聊什么、聊多深。（选项：互联网/金融/教育/医疗/学生/自由职业/其他 + 自定义） |
| **1.4 朋友们怎么形容你？** | 开放文本（≥10字） | 全新：这是最有效的问题之一。用户的自我认知+朋友视角=社交人格的锚点。提示："用3个词+理由" |
| **1.5 你的典型一天** | 开放文本（简短） | 全新：让克隆知道用户的生活节奏和日常语境。提示："随便说几个片段就行，比如'早上咖啡，晚上打球'" |
| **1.6 兴趣深度追问** | 选择+追问 | 升级原有：不只"你喜欢什么"，还要"为什么/怎么喜欢"。保留原来选择 1-4 个，但每选一个追加一行可选文本："你最近因为 X 感到充实的一次是？" |
| **1.7 一个改变了你的经历** | 开放文本（可选） | 全新：这是价值观和人格的"根"。一句真实经历比 10 个标签更能定义一个人。提示："不一定要很重大，改变了你想法的一件小事也行" |

**M1 对相似度的贡献**：
- 克隆知道用户的**生活语境**（职业、日常、城市），不会在对话中暴露"无知"
- 克隆能引用用户的**真实经历**作为谈资，而非编造
- 克隆的行为模式与用户的**人格自述**一致（朋友眼中的人 = 克隆表现的人）

### Module 2：语言指纹（Voice Print）

**目标**：让克隆拥有用户的表达方式——不是"说正确的话"，而是"用你的方式说话"。

这是当前系统相对做得最好的模块，但仍然太浅。核心问题是：**3 个场景选择题捕捉的是"用户想成为的样子"，不是用户真实说话的方式。** 自由文本才是真正的语音样本。

| 问题 | 类型 | 为什么重要 |
|------|------|-----------|
| **2.1 语气标签 + 证据** | 多选 + 每个标签配一句用户的真实原话 | 升级原有：仅选标签毫无意义——"幽默"在不同人身上完全不同。必须用证据锚定："我选了'幽默'，因为我经常说_________"。每个标签配一个真实例句 |
| **2.2 场景选择（扩充至 6 个）** | 选择题（每场景 3-4 个选项） | 升级原有：从 3 场景 → 6 场景，覆盖更多情绪：<br>**(a) 日常邀约** 周末干嘛（保留）<br>**(b) 观点分歧** 意见不同时（保留）<br>**(c) 破冰回复** "在吗"怎么回（保留）<br>**(d) 分享兴奋** "跟你说个好消息！"你怎么接？<br>**(e) 安慰朋友** 朋友失恋/受挫了，你怎么说？<br>**(f) 吐槽日常** 遇到无语的事，你怎么跟朋友说？ |
| **2.3 回复特定消息** | 开放文本（选 1-2 条回复） | 全新：**这是 M2 最重要的数据采集机制。**设计原则：**不要求用户表演自己，而是给出具体刺激让用户自然反应。**给用户看 3 条有具体情境、有情绪的真实感消息，选 1-2 条用平时口吻回复：<br>**(a) 吐槽场景** 朋友：「我服了，今天开会老板又画饼，说下季度肯定涨薪 🙄」<br>**(b) 兴奋场景** 朋友：「啊啊啊我抢到周杰伦门票了！！！！！」<br>**(c) 关心场景** 朋友：「有点烦，能聊聊吗？」<br>**为什么比"给朋友写条消息"好：**用户不需要发明场景、不需要模拟情绪——场景和情绪已经提供了，用户只做一件熟悉的事：**回复消息**。语气是自然反应的副产品，不是被要求表演出来的。 |
| **2.4 补全句子** | 开放文本（3 句填空） | 升级原有 sampleMessage：用 Mad Libs 式的填空降低创作门槛。给出句子开头，用户填完整："说实话我觉得______""哈哈那确实______""不是我说，______"。半结构化格式天然暴露惯用句式，且零表演压力。 |
| **2.5 粘贴真实消息（可选）** | 文本输入 | 全新：**唯一不经"问卷意识"过滤的样本。**提示："想得到最像你的克隆？从最近的聊天记录里复制 1-2 条你发给朋友的真实消息。我们只提取语气特征，不会保存具体内容。"□ 跳过，用上面写的就够了。这是数据质量最高的采集方式，但设为可选因为涉及隐私感受。 |
| **2.6 聊天习惯偏好** | 多选 | 全新：平时打字习惯——用不用句号？爱不爱分段？emoji 用得多不多？发不发语音条？这些细节决定了克隆的"打字感" |

**M2 对相似度的贡献**：
- **回复特定消息（2.3）** 是整个 M2 最有价值的采集方式：用户面对具体刺激做出自然反应，语气是副产品而非表演对象，覆盖了吐槽/兴奋/关心三种核心社交情绪
- **补全句子（2.4）** 是零摩擦的语气探针，半结构化的格式天然暴露惯用句式和连接词偏好
- **真实消息（2.5）** 是语气采集的"黄金样本"——用户不是在为克隆写东西，用户就是在聊天
- 口头禅（已纳入 2.4）是**克隆可信度的"作弊码"**——一句精准的口头禅比十段 AI 生成更像本人
- 6 场景（2.2）覆盖了**主要情绪类型**（开心/难过/生气/兴奋/无聊），让克隆在不同情绪下的表达都有据可依
- 证据锚定（2.1）解决了标签模糊问题——"幽默"不再是抽象概念，而是具体的说话方式

### Module 3：信念系统（Belief System）

**目标**：让克隆拥有用户的判断框架——在遇到问卷没有覆盖的话题时，能"按你的方式思考"。

这是当前系统的最大缺失：只有 2 道约会相关二选一。结果就是克隆在约会以外的话题上完全失锚。

| 问题 | 类型 | 为什么重要 |
|------|------|-----------|
| **3.1 关系观** | 选择题 + Why 追问 | 升级原有 pace 问题："你相信'对的人'是遇到的还是磨合出来的？" / "一段关系里，你最不能接受什么？" + "为什么？" |
| **3.2 分歧观** | 选择题 + Why 追问 | 升级原有 conflict 问题：不只问"你怎么做"，还要问"为什么这样"和"上一次这样做的结果是什么" |
| **3.3 信任观** | 选择题 + 开放文本 | 全新："你一般怎么判断一个人是否值得信任？"这是关系社交中最底层的判断逻辑，克隆必须知道 |
| **3.4 幸福观** | 开放文本（简短） | 全新："什么让你真正感到开心？"这不是约会问题，这是人格问题。一个人在意什么，由什么驱动 |
| **3.5 日常观点探针（3-4道）** | 选择题 + 可选 Why | 全新：不是约会话题，是日常聊天里真正会聊到的话题。例如：<br>- "你怎么看'努力就有回报'这句话？"<br>- "别人在社交媒体晒'完美生活'，你什么感觉？"<br>- "朋友借钱不主动还，你会怎么处理？"<br>- "你觉得一个人最难得的品质是什么？" |
| **3.6 改变过想法的事** | 开放文本（可选） | 全新：一个人"愿意改变想法"本身就是重要的人格特征。而且这暴露了用户的认知弹性度 |

**M3 对相似度的贡献**：
- 信念系统是克隆的**默认决策引擎**。当遇到陌生话题时，克隆不是随机生成观点，而是从价值观出发推导——这才是"像你"的底层逻辑
- "为什么要这样"比"你选了什么"重要 10 倍。选择告诉你 WHAT，理由告诉你 WHY。克隆用 WHY 来应对新情况
- 日常观点探针（3.5）覆盖了**最可能出现在闲聊中的话题类别**，让克隆在这些场景下不会"失语"

### Module 4：关系地图（Relational Map）

**目标**：让克隆知道如何根据对话对象调整自己——真实的人在不同关系中说话方式不同。

这是当前系统完全缺失的。当前克隆对所有人说话方式一样，这在社交场景中是致命的漏洞。

| 问题 | 类型 | 为什么重要 |
|------|------|-----------|
| **4.1 社交光谱** | 滑块或选择 | 全新：在三个维度上标记自己——<br>- 和陌生人：拘谨 ← → 自来熟<br>- 和朋友：倾听者 ← → 分享者<br>- 在群体中：观察者 ← → 气氛组 |
| **4.2 被理解的信号** | 开放文本 | 全新："别人怎么做/怎么说，会让你觉得'他懂我'？"这告诉克隆在对话中应该朝哪个方向努力 |
| **4.3 让我不想说话的触发** | 开放文本 | 全新："别人说什么/做什么，会让你突然不想聊了？"这是克隆需要避免的行为模式 |
| **4.4 表达关心的方式** | 选择 + 文本 | 全新："你一般怎么对在乎的人表达关心？"（直接说/默默做事/分享好东西/调侃）克隆需要知道用户的"爱的语言" |
| **4.5 情绪反应模式** | 2 道场景题 | 全新：<br>- "心情不好时，你希望别人______"（陪我聊聊 / 给我空间 / 逗我开心 / 不用管我）<br>- "特别开心时，你会______"（立刻分享 / 自己消化一会 / 请客庆祝 / 发朋友圈） |

**M4 对相似度的贡献**：
- 关系地图赋予了克隆**情境感知能力**——在"刚认识的人"和"老朋友"之间切换表达方式
- 这可能是**用户感觉"不像"的最大单一原因**：如果克隆对所有匹配对象说话都一个调调，用户会觉得"这不是我"
- 情绪反应模式让克隆在关键社交时刻（安慰、庆祝、冲突）有据可依，不至于说出让用户尴尬的话

### Module 5：深度对话（Deep Dialogue）

**目标**：用非结构化的方式补齐结构化问卷的盲区，暴露矛盾，获取真实的语言样本。

**当前对话的问题**：
- 方向随机，依赖用户主动找话题
- 对话太短（4-8轮），样本量不足以做语言学分析
- AI 没有利用前面问卷的信息来追问

**改进后的设计**：

| 改进点 | 具体做法 |
|--------|---------|
| **AI 角色转变** | 从"随便聊聊的朋友"变为"好奇的采访者"。AI 拿到了前 4 个模块的答案，其任务是有针对性地追问 |
| **对话结构** | 分四个阶段：<br>**(a) 暖场（1-2轮）**：轻松话题，让用户进入状态<br>**(b) 矛盾追问（2-3轮）**：AI 发现用户回答中的内在矛盾并提问。例如："你选了'直接'，但在分歧场景你选了委婉……这两种情况对你来说有什么不同？"<br>**(c) 深层话题（2-3轮）**：基于用户兴趣/经历展开。例如："你提到了那次改变你想法的事，当时是什么感觉？"<br>**(d) 收尾（1-2轮）**：轻松收尾，为孵化铺垫 |
| **轮数增加** | 4-8轮 → 8-12轮（最小 6 轮，推荐 10 轮） |
| **离线兜底** | 保留当前 `OFFLINE_PROMPTS` 机制，但每句问法升级为"基于已有答案的个性化追问" |
| **语言学分析** | 在 finalize 时，额外将对话记录送入 LLM 做语言学特征提取：句长中位数、语气词频率、emoji 密度、回复长度偏好、主动/被动比例 |

**M5 对相似度的贡献**：
- 对话是**唯一能捕捉到"结构化问题问不到的东西"的模块**——用户在自由对话中暴露的口语习惯、思维跳跃、情绪反应
- 矛盾追问（b 阶段）捕捉到的是**真实的复杂性**——人本来就不一致，克隆也应该保留这种不一致
- 大幅增加的轮数提供了足够的**语言学分析样本**，让 style.md 生成不再是"猜"，而是"分析"

## 四、为什么这套方案能保证克隆相似度？

### 4.1 理论框架

一个数字分身的"像"，建立在它能答对三类问题：

```
第一类："你知道 X 吗？"        → 知识覆盖 → M1 提供的
第二类："你会怎么说 X？"       → 表达复制 → M2 提供的
第三类："你怎么看 X？"         → 判断对齐 → M3 + M4 提供的
```

当前系统只勉强覆盖第一类和第二类，而且覆盖很浅。重构后的五层模型同时覆盖三类，并且每类的深度都大幅提升。

### 4.2 关键逻辑链

```
M1（身份基座） → 克隆知道你是谁
M2（语言指纹） → 克隆说话像你
M3（信念系统） → 克隆用你的方式思考
M4（关系地图） → 克隆对不同人像不同的"你"
M5（深度对话） → 填补前四个模块的盲区 + 捕捉矛盾 + 提供真实语言样本
```

### 4.3 一个具体的对比

**场景：匹配对象在深夜发了一条"今天好累啊"**

| | 当前问卷产出的克隆 | 五层模型产出的克隆 |
|------|-------------------|-------------------|
| **能回复** | ✅ 会回复 | ✅ 会回复 |
| **语气像用户** | ⚠️ 部分像（语气标签起作用） | ✅ 像（有真实对话样本 + 语言学分析） |
| **回复方式符合用户习惯** | ❌ 可能太长/太短/太热情/太冷淡 | ✅ 知道用户是"直接关心"还是"调侃转移"还是"分享自己的累" |
| **知道用户会怎么想** | ❌ 不知道用户对"累"的态度 | ✅ 从 M1 日常习惯 + M3 幸福观 + M4 表达关心的方式中推导 |
| **对什么类型的人说什么话** | ❌ 一视同仁 | ✅ 从 M4 关系地图判断对方的亲密度，调整回复 |

### 4.4 数据量对比

| 指标 | 当前 | 重构后 |

| 结构化数据字段 | ~8 个 | ~25+ 个 |
| 自由文本样本字数 | 0-30 字（sampleMessage） | 150-400 字（回复消息 + 补全句子 + Why 追问 + 对话） |
| 对话轮数 | 4-8 轮 | 8-12 轮 |
| 覆盖的情绪场景 | 3 种 | 8+ 种 |
| 价值观/观点覆盖 | 仅约会 2 题 | 约会 + 日常 8+ 题 |
| 关系维度 | 0 | 5 个维度 |

## 五、实施建议

### 5.1 UX 设计原则

1. **不要一口气全给。** 保持"每屏一个问题"的节奏，但每个问题都更值得回答。
2. **必选 vs 可选分层。** M1-M3 中的开放文本问题设为"可选但强烈推荐"，M4 设为可整体跳过。
3. **进度可见。** 显示"5个模块，当前第 X 个"而不是"步骤 3/12"。
4. **给即时反馈。** 用户每完成一个模块，展示一个小摘要："我记住了——你喜欢 X，用 Y 的方式说话，相信 Z"。
5. **不要求用户表演自己。** 语气采集类问题遵循"刺激→反应"模型：给用户一个具体的情境和刺激（一条有情绪的消息、一个句子开头），让用户**自然反应**，语气是副产品。永远不要问"请用你的语气说一段话"——这会让用户自我审视，自我审视是真实语气的杀手。

### 5.3 style.md 生成升级

当前 `StyleGeneratorService` 只分析 tone + few-shots。升级后应额外分析：

1. **语言特征量化**：从句长分布、语气词密度、emoji 使用模式中提取可验证的约束
2. **矛盾标记**：在 style.md 的 Avoid 部分新增 "Contradictions"，例如"大部分时候直接，但触及 X 话题时会绕弯"
3. **关系切换模式**：在 style.md 新增 "Adaptation" 章节，描述用户在不同亲密度下的语气切换规则

### 5.4 实施优先级

| 优先级 | 模块 | 理由 |
|--------|------|------|
| P0（必须） | M2 语言指纹升级 | 回复消息采集 + 补全句子 + 6场景：这是"像不像"最直接的决定因素 |
| P0（必须） | M3 信念系统 | 日常观点 + Why 追问：填补当前最大盲区 |
| P1（高优） | M1 身份基座扩展 | 经历 + 自我认知 + 日常：大幅提升克隆的"知识丰富度" |
| P1（高优） | M5 深度对话升级 | 导向式追问 + 矛盾捕捉：提升对话样本质量 |
| P2（可延后） | M4 关系地图 | 对匹配质量影响大，但对基础"像不像"影响相对小 |

### 5.5 最小可行验证

如果你想在全面实施前验证这套思路，可以用 2 个测试用例先跑：

1. **对比测试**：找一个用户分别用旧问卷和新问卷创建分身，让 3 个了解该用户的朋友做盲测，判断哪个克隆更像。
2. **压力测试**：对克隆问 5 个问卷未覆盖的问题（如"你怎么看远程办公？""你相信星座吗？"），对比旧克隆和新克隆的回答哪个更像用户本人。

## 六、问答：设计背后的取舍

**Q: 问题太多了，用户会中途放弃怎么办？**

A: 两个策略：一是所有开放文本问题都标为"可选"，降低完成门槛；二是"快速通道"——用户可以在任意模块选择"跳过这个模块"，克隆会用合理的默认值填充。M1-M3 各有 1-2 个必选问题，其余都可选。

**Q: M4 关系地图真的重要吗？这是一个约会应用，大部分匹配刚认识。**

A: 正因为大部分匹配刚认识，关系地图才重要——它告诉克隆"对刚认识的人你应该怎么说话"。没有 M4，克隆对所有人一视同仁，这在约会场景是致命的。一个人对陌生人、暧昧对象、好友的语言策略完全不同。

**Q: 为什么不直接让 AI 和用户聊 30 分钟就完事了？**

A: 30 分钟自由对话确实能采集很多数据，但它依赖用户主动暴露关键信息——大多数人在闲聊中不会主动说"我信任一个人的标准是____"或"我改变想法的一次经历是____"。结构化问卷 + 对话追问的组合，才能同时覆盖"用户知道自己是什么样"和"用户没意识到但对话暴露出来的"两方面。
